# 캐릭터·감성 요소 도입 방향성 (v1.0)

> 목표: "심플하면서 감각적이고 귀엽고 재미있는" 캐릭터/요소를 앱에 도입하기 위한 벤치마킹 기반 방향성 정의
> 대상: 악보 배우기 — 무빌드 바닐라 JS PWA (모바일 우선, 라이트/다크, ko/en)
> 작성 방법: **더블 트랙 크로스 체크** — ① Claude 6개 관점 웹 벤치마킹(Duolingo·음악 학습 앱·한국 캐릭터 트렌드·키즈 교육 앱·마이크로인터랙션·무빌드 구현 기술) ② Codex 독립 코드 대조 분석 ③ 3개 렌즈(심플함·구현 가능성·재미) 비판 검증. 양측이 독립적으로 합의한 결론은 신뢰도 높음으로 표기. 크로스 체크 상세는 §8.

## 0. 현행 자산 감사 — "무엇이 이미 있는가"

방향성 논의 전 필수 전제. 초기 분석에서 "시각 리워드 전무"로 오판했으나 코드 대조로 교정됨:

| 자산 | 위치 | 상태 |
|---|---|---|
| 콤보 카운터 + 콤보 보너스 점수 | `game.js` `submit()` — combo/maxCombo, 최대 5배 보너스 | ✅ 구현됨 · ❌ 퀴즈 중 화면에 안 보임 |
| 별점 1~3성 (100%/85%/70%) | `game.js` `stars()`, 결과 화면 ★☆ 렌더 | ✅ 구현됨 · ❌ 세션 종료 시 휘발, 홈에 안 보임 |
| 결과 화면 리워드 | `ui.js` `renderResult()` — 🎉/👏 이모지, 별점, 정확도 게이지, 점수·최고 콤보, 오답 리뷰 | ✅ 구현됨 |
| 정답/오답 연출 | `.pop`/`.shake` keyframes, ✓/✗ 기호, 정답음→실제 음 순차 재생 | ✅ 구현됨 |
| 접근성 기반 | `prefers-reduced-motion` 블록, aria-live 피드백, 색+기호 이중 신호 | ✅ 구현됨 |
| 진행 저장 | `storage.js` — 설정 + `learnSeen`만 영속. 별점·점수·콤보는 휘발 | ⚠️ 부분 |

**진짜 갭 4가지**: ① 캐릭터/감정 표현이 없다 ② 성과가 누적되지 않는다(영속화 없음) ③ 콤보가 퀴즈 중 보이지 않는다 ④ 홈에 "다시 올 이유"가 없다.
→ 따라서 이번 작업의 본질은 **"새 게임화 시스템 신설"이 아니라 "이미 있는 값에 감정과 기억을 입히는 것"**이다.

## 1. 목표 — 정직한 근거 위에서

교육 에이전트 연구(메타분석)는 캐릭터가 **학습 성과가 아니라 정서 경험·동기**에 효과가 있음을 보여준다. Duolingo의 리텐션도 스트릭·목표 장치가 주도하고 캐릭터는 그 위의 감정 레이어다. 따라서 목표를 두 축으로 분리한다:

- **축 1 — 캐릭터(감정 채널 + 브랜드)**: 정답·오답·완주의 순간에 감정적 온기를 입히고, 파비콘부터 결과 화면까지 하나의 조형 언어로 통일
- **축 2 — 누적·목표 장치(재방문 동기)**: 별점 영속화, 주간 기록, 소품 성장 — 캐릭터와 별개 축으로 로드맵에 병렬 배치

"캐릭터를 넣으면 사용자가 돌아온다"는 인과는 어떤 벤치마크로도 입증되지 않는다 — 두 축을 섞지 말 것.

## 2. 디자인 원칙 — "절제가 곧 감각"

| # | 원칙 | 근거 |
|---|---|---|
| P1 | **문항 풀이 중 장식 캐릭터 상주 금지.** 캐릭터는 피드백·결과·홈 등 "쉼표 구간"에만. 단 **기능 정보(콤보 카운터 등)는 예외** — 장식이 아니라 진행 상태다 | seductive details 연구, Kahoot |
| P2 | **빈도 반비례 법칙.** 매 문항=300ms 마이크로, 5연속 콤보=1초, 퍼펙트·첫 완주=컨페티급. 자주 나올수록 작게. **한 화면에 감성 장치 최대 1개** | over-confetti UX 연구, NN/g |
| P3 | **오답 = 정보이지 벌이 아님.** 캐릭터가 슬퍼하는 연출 금지. 갸웃 + 정답 안내 + 응원 톤까지만 | Todo Math, Duolingo |
| P4 | **조형은 빼기로.** 기하 도형 2~4개 + 큰 눈 + 최소 이목구비. **신규 색 도입 대신 기존 토큰(--brand/--bad/--hint) 재사용 원칙**, 캐릭터 전용 토큰은 2~3개 상한 | 카카오프렌즈, Toss Face, baby schema |
| P5 | **귀여움 수위는 "미니멀+위트"로 연령 중립.** 유아 톤(반짝이 과다, 아기 말투, '-이/친구/패밀리' 네이밍 남발) 배제 | minini·산리오의 연령 중립, 성인 입문자 포함 |
| P6 | **모든 연출은 기능에 얹는다.** 정보 없는 랜덤 인사말, 목적 없는 상시 애니메이션 금지. 말풍선은 실제 정보(이어서 학습, 신기록)가 있을 때만 | NN/g "delight는 목적이 될 수 없다" |
| P7 | **접근성·성능 우선.** transform/opacity만, reduced-motion 존중(장식만 끄고 상태 피드백 유지), 장식 SVG는 aria-hidden, ✓/✗ 기호 신호 유지 | WCAG C39·1.4.1 |
| P8 | **표정 예산과 포즈 예산 분리.** 표정(얼굴 파츠) 5종 상한: idle / happy / cheer / hmm / sleepy. 포즈는 transform 애니메이션(hop·갸웃·끄덕임)으로 해결 — 손이 필요한 동작(엄지척·만세)은 조형 문법 위반으로 금지 | 무빌드 유지보수 상한 |

## 4. 캐릭터 컨셉

**공통 전제**: 기존 IP(부엉이·게 등) 형태 모방 금지. **퍼블릭 도메인인 음악 기호에서 형태를 유도** — 저작권 리스크 제로 + 도메인 정합성 + "학습 대상=캐릭터"라 주의 분산 문제가 구조적으로 없음. *(양 트랙 독립 합의 — 신뢰도 최상)*

### A안. 음표 마스코트 1종 ★ 채택

- **컨셉**: 8분음표(♪)의 머리(notehead)를 얼굴로 쓰는 단독 마스코트. Claude 트랙("온음이")과 Codex 트랙("콩음이")이 **독립적으로 동일 결론**에 도달한 안.
- **성격**: 호기심 많고 반응이 빠른 작은 음표. 잘하면 폴짝 뛰고, 틀리면 같이 갸웃하며 힌트 쪽을 가리킨다. 가르치는 선생님이 아니라 **같이 듣고 확인하는 동료**.
- **비주얼**: 타원(머리) + stem·flag는 **기보 원형 그대로 유지**(변형하면 기호 인지를 흐림 — 검증 렌즈 지적 반영) + 낮고 넓게 배치한 점 눈 2개 + 미세 기울기. 입은 평소 없거나 최소. SVG 요소 10~15개, viewBox 하나로 32px(앱바)~120px(결과) 스케일링.
- **컬러**: 몸통 `--brand` 계열, 눈·윤곽은 전용 토큰 `--mascot-ink`(피드백 영역의 currentColor 오염 방지), 볼터치는 다크 모드·16px 검증 후 결정하는 **옵션**.
- **네이밍**: 미확정 — §10 오픈 결정. '-이/친구' 애칭은 유아 레지스터 + 영어 대응 불가로 배제하고, ko/en 공용 후보(도도 Dodo, 미미 Mimi, 솔라 Sola 등 계이름 유래)에서 성인 스크리닝 후 확정.

### B안. 별 리워드 그래픽 (보조 채택)

- 별점·결과 화면 전용의 반짝임 그래픽(Codex 제안 "마디별"). 얼굴·성격 없는 순수 리워드 레이어라 UI를 방해하지 않음. 기존 ★☆ 별점의 시차 팝 연출에 흡수.

### C안. 화음 블록 / 보표 서브 그래픽 (P2 전용)

- 루트·3도·5도가 쌓이는 블록(화음 모드 교육 그래픽), 모드별 소품(음자리표 깃발 등). **새 캐릭터를 늘리지 않고 A안의 조형 문법 + 소품으로만 확장** — 캐릭터 라인업 상한 1종은 양 트랙 공통 결론.

## 5. 적용 지점 매핑 (실제 코드 위치 기준)

| 화면 | 코드 위치 | 넣을 것 | 넣지 말 것 |
|---|---|---|---|
| **홈** | `ui.js` `renderMenu()` — 타이틀 영역 (앱바 중앙은 세그먼트 컨트롤이 이미 점유) | 타이틀 옆 마스코트 1마리(idle 깜빡임). 말풍선은 정보 있을 때만(이어서 학습 제안·신기록). P1: 카드에 최고 별 배지 | 카드마다 캐릭터 반복, 미학습 시 우는 표정(죄책감 금지), 랜덤 인사말 |
| **사전 학습** | `learn.js` `renderLearn()` 타이틀 아래. 각 learn 함수는 로컬 render()라 정적 SVG가 안전 | 레슨 도입부 소형 마스코트 + 핵심 개념 1줄 말풍선. 모드별 포즈(줄/칸 가리키기 등) | 오선 위 음표에 눈 붙이기(기보 학습 전이 방해 — 오선보는 학습·퀴즈 모두 항상 실제 기보 형태), 상시 애니메이션 |
| **퀴즈 풀이 중** | `ui.js` `renderMode()` | **장식 캐릭터 없음.** P1: 피드백 영역에 콤보 카운터 텍스트("🔥 3연속") — 기능 정보 | 상주 마스코트, 오선보·선택지와 겹치는 모든 장식 |
| **정답 피드백** | `ui.js` `onChoose()` — feedbackEl. 현재 textContent 교체 방식이라 DOM 구조 변경 + min-height 공간 예약 필요(CLS 방지) | 소형 마스코트 happy + squash&stretch 원바운스(300ms). 문구는 **현행 1~2단어 유지**(850ms 자동 진행 내 읽기 한계). 5연속 이상만 윙크+음표 파티클 4~6개 | 매 정답 컨페티, 850ms를 막는 연출, 읽을 수 없는 긴 칭찬문 |
| **오답 피드백** | 동일 (1700ms) | 갸웃(rotate 4deg) + 중립~응원 표정 + 힌트 자동 오픈(`hintBox.open`)을 가리키는 연출로 "실패"→"복습" 전환 | 슬픈 표정, ✗ 기호 제거(색각이상 대응 — WCAG 1.4.1), shake 과다 |
| **결과 화면** | `ui.js` `renderResult()` — `result-emoji`(🎉/👏)를 마스코트로 교체 | 정확도 3구간 표정(만점=cheer / 70%+=happy / 미만=응원 hmm + "다시 하면 늘어요"). **만점·신기록·모드 첫 클리어에만** 음표(♪) 컨페티 2초. 별점 시차 팝, 점수 카운트업+게이지 오버슈트 | 모든 완주에 컨페티(가치 소진), 낮은 점수에 실망 연출 |
| **설정** | `ui.js` `renderSettings()` | **"캐릭터 표시" 토글(기본 켜짐).** off 시 마스코트·말풍선·파티클만 제거되고 색·카운트업·게이지 등 비캐릭터 피드백은 유지됨을 명세 | 컨트롤 터치 영역 침범 |
| **파비콘·PWA 아이콘** | `index.html`(인라인 SVG), `icons/*.png`, `sw.js` ASSETS | P0: 파비콘(SVG)만 마스코트 얼굴로 교체. PNG 아이콘 4종은 P1(래스터라이즈 공정 필요 + 기설치 PWA는 미갱신 한계 명기) | 세부 묘사 많은 전신 아이콘, 잦은 교체 |

## 6. 무빌드 구현 전략

### 6.1 SVG 캐릭터
- 기존 `staff.js`와 동일하게 **`createElementNS` 기반 ES 모듈** `src/mascot.js` — `createMascot({ mood, size })` 반환. `sw.js` ASSETS에 추가 필수(오프라인 로드).
- 표정 파츠(눈·눈썹·입)는 전부 겹쳐 두고 루트 `data-mood`로 **opacity 스왑** — 이것이 유일 표준(SMIL은 유지보수 중단 상태 + reduced-motion 미존중으로 배제). 표정 5종 상한.
- 고정 치수(`width/height` CSS 명시)로 레이아웃 흔들림 방지. 장식 SVG는 전부 `aria-hidden="true"`.

### 6.2 CSS 애니메이션
- **transform/opacity만.** SVG 내부 요소에는 `transform-box: fill-box; transform-origin: center` 필수(누락 시 뷰포트 기준으로 파츠가 날아감).
- 눈 깜빡임: `@keyframes blink { 0%, 92%, 100% { transform: scaleY(1) } 96% { transform: scaleY(0.1) } }` 4s infinite (키프레임 3개 필수 — 1개면 서서히 감김).
- 정답 hop: squash & stretch(착지 `scale(1.15, 0.85)` → 도약 `scale(0.9, 1.1)`), `animationend`에서 클래스 제거.
- 파티클: 콤보용은 DOM span 4~6개 + CSS 변수 `--dx/--dy`, 결과 컨페티는 canvas(60~80개 상한, `ctx.fillText('♪')`, 종료 시 정리). 오선보 SVG 자체는 절대 움직이지 않음.

### 6.3 재렌더 아키텍처 주의 (이 코드베이스 특유의 함정)
- `setState`마다 `app().replaceChildren()`로 **화면 전체가 재생성**됨 → 문항 간 CSS transition 연속성 불가(진행 도트에 이동 연출 넣기 금지), 홈 idle 애니메이션이 토글 조작마다 재시작됨.
- 홈 마스코트 탭 인터랙션 등 국소 갱신은 **기존 soundBtn 패턴(setState 우회 직접 DOM 갱신)**을 따를 것. 재렌더 점프 버그(B1) 이력이 주석으로 남아 있음.

### 6.4 다크 모드·접근성
- 신규 hex 하드코딩 금지. `--mascot-body`/`--mascot-ink` 토큰 2~3개만 light/dark 양쪽에 추가, 다크는 채도↓ 명도↑ 튜닝 + AA 대비 검증.
- reduced-motion: 기존 opt-out 블록(`.pop`/`.shake` 무효화)에 신규 클래스 추가 — **장식 모션만 끄고 표정·색 상태 피드백은 유지**(WCAG C39). JS는 트리거 시점에 `matchMedia` 조회(1회 캐시는 OS 설정 변경 미반영).
- 말풍선 텍스트는 장식이 아닌 콘텐츠 → 실제 텍스트 노드로 렌더하되 기존 aria-live 피드백과 중복 발화 방지 설계.

### 6.5 보상 데이터 저장 정책
- 현행 `storage.js`는 "스키마 변경 시 리셋 허용" 정책 — **획득 자산(별점·소품)에는 부적합.** 보상 전용 키 `sing-rewards-v1` 신설 + "필드 추가만 허용, 삭제·리셋 금지" 가산적 스키마 정책. 리셋 허용은 `learnSeen` 같은 저가치 데이터에만 한정.

### 6.6 사운드·i18n
- 기존 WebAudio(`playCorrect`/`playWrong` + 정답 음 순차 재생) 타이밍에 시각 연출을 맞춤. **사운드 절제 원칙**: 특수음은 세션당 상한, 기존 톤과 음색 통일, `audioEnabled` off 시 시각 피드백 단독으로 성립해야 함.
- 캐릭터 대사는 `i18n.js`에 ko/en 동시 설계. 현행 `t()`는 평면 딕셔너리라 랜덤 문구 배열·템플릿 보간은 소규모 확장 필요.
- **보이스&톤 가이드**: 해요체, 문장당 느낌표 최대 1개, 아기 말투("~했쪄요")·과도한 이모지 금지, 1문장 이하. 캐릭터가 학습 설명을 대신하지 않음.

## 7. 단계별 로드맵

> 재조정 원칙(검증 렌즈 반영): P0에 **wow 모먼트 1개 + 누적 장치 1개**를 반드시 포함 — 첫 세션에 감정 피크와 "내일 또 올 이유"를 보장.

### P0 — 최소 매력 (규모: 중)
1. 마스코트 SVG(`mascot.js`) + 표정 5종 + 다크 토큰 + aria-hidden + reduced-motion 가드 + sw.js 등록 — *전체 공수의 최대 항목: 시안 2~3종 → 16px·다크 검증 → 손코딩 이관*
2. 정답/오답 피드백 리액션(hop 300ms / 갸웃) — 피드백 DOM 구조 변경 + min-height 예약 포함. 문구는 현행 유지
3. 결과 화면: `result-emoji` → 3구간 표정 교체 + **만점·첫 완주 한정 음표 컨페티** (wow)
4. **별점·최고점 영속화(`sing-rewards-v1`) + 홈 카드 최고 별 표시** (누적)
5. 파비콘(SVG) 교체
6. 설정 "캐릭터 표시" 토글(off 폴백 명세 포함)

### P1 — 반응성·리듬 (규모: 중)
1. 퀴즈 중 콤보 카운터 표시(기능 정보) + **5연속 이상만** 윙크+파티클(3연속은 사운드 피치 신호만 — 4지선다에서 3연속은 너무 흔함)
2. 결과 별점 시차 팝 + 점수 카운트업 + 게이지 오버슈트
3. **세션 멜로디**: 정답 음을 누적해 결과 화면에서 "오늘 만든 멜로디" 재생 — 음악 앱만 가질 수 있는 도메인 고유 보상(WebAudio 기반, 저비용) + 마스코트 오디오 시그니처(2음 차임)
4. 사전 학습(L-1~L-8) 말풍선 가이드 + 모드별 포즈
5. 주간 학습 도트 7개(깨짐 없는 누적형) + **목표 연결**(주 3일 달성 → 소품 획득) + 학습 완료 시 홈 표정 전환
6. 홈 마스코트 탭 인터랙션(soundBtn 패턴) · PWA PNG 아이콘 4종 교체(래스터라이즈 공정 명시)

### P2 — 수집·성장 (규모: 대, A안 검증 후에만)
1. **마스코트 소품 성장**(모자·목도리·안경 — SVG 파츠 1~2개씩): 수집물을 캐릭터에 입혀 정서 앵커와 누적 동기를 결합. *수집 메커니즘은 이것 하나만 — 별도 "정원" 화면 등 제2 수집 레이어 금지(절제 렌즈 반영)*
2. 화음 블록·모드 소품(C안) — 모드 마스터와 1:1 대응
3. 소수 정예 배지 6~8종("첫 퍼펙트", "조표 정복" 등 구체 성취만)
4. 오답 재출제 복습 루프(wrongLog 활용, 코드 주석에 백로그로 명시된 항목) + "지난번에 틀렸던 문제를 맞혔어요" 과정 칭찬
5. (검토) 용서형 스트릭 / 마스코트 이름 짓기(input 1개 + localStorage, 애착 효과 큼) / 햅틱 `navigator.vibrate()`

**단계 완료 게이트**: 16px 판독 / 다크 AA 대비 / reduced-motion / 850·1700ms 타이머 비간섭 / ko·en 문구 완비 / 캐릭터 off 상태 정상 동작 / 콤보 연출 빈도 시뮬레이션(문항 수 옵션별).

## 8. 크로스 체크 결과 (Claude ↔ Codex)

### 독립 합의 — 신뢰도 최상
- **음표 의인화 마스코트 1종**으로 시작 (양측 1순위 동일)
- 캐릭터는 주인공이 아닌 **피드백 레이어** — 주인공은 음의 위치·이름·소리
- 퀴즈 풀이 중 비상주, 오선보 영역 불변, 기존 사운드 타이밍에 동기화
- 결과 화면(`result-emoji`)이 캐릭터 효과 대비 비용이 가장 낮은 무대
- 인라인 SVG + 기존 토큰 재사용 + aria-hidden + reduced-motion + sw.js 갱신
- 캐릭터 라인업 억제 — 새 캐릭터 대신 **소품 확장**

### 각 트랙의 고유 기여
- **Codex**: 실제 파일·함수 위치 매핑, **기존 자산(콤보·별점·결과 연출) 존재 확인** → Claude 초안의 "신설" 오류 교정, learn.js 로컬 render 구조·설정 화면 터치 영역 등 통합 주의점
- **Claude**: 출처 있는 벤치마크 근거 체계, 절제 원칙의 명문화, 세션 멜로디·소품 성장·과정 칭찬 등 동기 설계 아이디어, transform-box 등 구현 함정 선제 식별

### 이견과 해소
| 쟁점 | 해소 |
|---|---|
| Claude 초안: 콤보·별점 시스템 "신설" | **오류 — 기존 자산의 연출·영속화·노출로 재정의** (Codex + 코드 대조로 확정) |
| Claude 초안: 학습 페이지 "표정 있는 음표" | **폐기** — 기보 학습 전이 방해 (검증 렌즈) |

## 9. 리스크와 Don'ts

### 리스크
| 리스크 | 완화책 |
|---|---|
| 성인 입문자가 유치하다고 느껴 이탈 | 미니멀+위트 톤, 설정 토글, 네이밍 성인 스크리닝, 토글 off 비율 추적 |
| 캐릭터가 인지부하를 높여 학습 저해 | 풀이 중 미등장 원칙 코드 주석으로 계약화 |
| 무빌드 유지보수 폭발 | 표정 5종·캐릭터 1종·토큰 3개 상한 |
| 보상 데이터 리셋으로 이탈 | `sing-rewards-v1` 가산적 스키마 (§6.5) |
| 연출 남발로 보상 가치 소진 | 빈도 반비례 + 화면당 1개 + 콤보 임계 5연속 |
| "스티커 붙인 느낌" (기존 UI와 조형 분리) | 시안 단계에서 기존 카드·라운딩·블루 토큰과의 정합 검토 선행 |

### Don'ts
1. 퀴즈 풀이 중 장식 캐릭터 상주 (기능 정보인 콤보 카운터는 예외)
2. 오답 시 우는/실망하는 캐릭터
3. 죄책감형 스트릭·시무룩 압박 (Duolingo guilt-trip은 성인 밈 문화용)
4. 화폐·상점형 보상
5. 배지 남발 (6~8종 소수 정예)
6. 기존 IP 형태 모방 (원칙만 배우고 형태는 음악 기호에서)
7. 매 정답 컨페티, 목적 없는 모션, 정보 없는 랜덤 인사말
8. 오선보·선택지 영역과 겹치는 장식, 오선 위 음표 변형
9. ✓/✗ 기호 신호 제거 (색각이상), aria-live 간섭, reduced-motion 무시
10. 한쪽 언어만 하드코딩된 대사, 아기 말투

## 10. 오픈 결정 사항 (구현 착수 전 확정 필요)

1. **네이밍**: ko/en 공용 후보 3안(예: 도도 Dodo / 미미 Mimi / 솔라 Sola — 계이름 유래로 용어 정확성 확보) 중 선택. '-이/친구' 계열 배제.
2. **정답 자동 진행 850ms 유지 vs 1200ms 연장**: 연출 여유 vs 앱 리듬 전체 영향 — 타이머 변경은 별도 의사결정.
3. **볼터치 포함 여부**: 귀여움 강도를 팔레트 레벨에서 올리는 결정 — 시안 + 다크·16px 검증 후.
4. **시안 방향**: 기호 원형 보존형 vs 부분 변형형 2안 병행 스케치 후 비교.
