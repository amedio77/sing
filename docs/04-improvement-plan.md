### 2.3 설계 — 세션 내 무중복 출제 + 풀 확장

**범위 선언**: 본 영역은 세션 내 중복만 다룬다. 세션 간(연속 재도전) 중복은 localStorage 도입(v0.8) 이후의 후속 범위로 백로그에 남긴다(§7).

**(1) 문제 정체성 키** — question 규약에 `key: string` 1필드 추가. 기준: **"해당 모드가 평가하는 학습 항목이 같으면 같은 key"**.

| 모드 | key 형식 | 예시 | 근거 |
|---|---|---|---|
| A | `${clef.id}:${english}${octave}` | `treble:E4` | 위치 읽기가 학습 항목 → 옥타브 포함 |
| B | `${dir}:${letter}` | `ko2en:C` | 글자 매칭이 학습 항목 → 옥타브(표시용) 제외 |
| C | `${sign}:${idx}` | `#:3` | next/nth는 같은 사실(SHARP_ORDER[idx])의 문구 변형 → **type 제외**로 같은 사실은 세션당 1회만 |
| D | `${dir}:${target.id}` | `notes2name:Cm7` | `target.id`는 chords.js에 이미 존재 |

**(2) 샘플링 — generate 재시도 + 세션 used-set (채택)**. 셔플백(전체 열거)은 무중복 보장이 더 강하지만 모드 계약(generate 1개 생성)을 pool() 열거로 바꾸는 큰 리팩토링이라 기각. 재시도 방식은 모드 변경이 key 1줄뿐이다. **단, 이는 수학적 보장이 아닌 확률적 회피다** — 시뮬레이션 20만 회 기준 풀 ≥ 14에서 중복 ≈0(관측치 0.0001 수준)이며, 완전 보장이 필요해지면 그때 셔플백으로 전환한다(YAGNI).

```js
// game.js createQuiz 내부 — questions 생성만 교체 (submit/accuracy/stars 등 불변)
const MAX_RETRY = 20; // generate는 순수 객체 생성(µs 단위), 스테이지 시작 시 1회뿐
const used = new Set();
const questions = [];
for (let i = 0; i < total; i++) {
  let q = generate();
  for (let r = 0; r < MAX_RETRY && q.key && used.has(q.key); r++) q = generate();
  if (q.key && used.has(q.key)) {
    // 풀 < 문항 수(예: A 쉬움 9 < 10): 중복 불가피 → 직전 문항과의 연속 중복만 금지
    const prevKey = questions[i - 1] && questions[i - 1].key;
    for (let r = 0; r < MAX_RETRY && q.key === prevKey; r++) q = generate();
  }
  if (q.key) used.add(q.key);
  questions.push(q);
}
```

폴백 규칙: ① used-set 회피 20회 → ② 실패 시 중복 허용하되 직전 key 회피 20회 → ③ 그래도 실패하면 수용(무한루프 원천 차단, 발생 확률 사실상 0). `key`가 없는 question은 dedup을 건너뜀(향후 모드가 key를 빠뜨려도 오류 없이 동작 저하만).

**(3) 모드 B 쉬움 양방향 확장 (풀 7→14)**: `modes.js:79`의 쉬움 `ko2en` 고정을 `pick(['ko2en','en2ko'])`로 통일. `labelFor`가 이미 양방향을 지원하므로 추가 변경 없음. 이 확장으로 B 쉬움 중복은 완전 소멸(14 ≥ 10). *쉬움 난이도 정의를 바꾸는 제품 결정이므로 독립 커밋으로 분리해 단독 롤백 가능하게 한다.*

**(4) 범위 제외 결정** (검증 결과 반영):
- **모드 C "조표 개수→장조" 유형 추가는 제외** — (2)+(3)만으로 전 모드 중복 0이 달성되어 반복 불만 해소에 불필요(YAGNI). 미사용 데이터 `SHARP_KEYS`/`FLAT_KEYS`·`KEY_RULES`를 활용하는 이 유형은 **학습 콘텐츠 확장**으로서 백로그에 등재(§7). 부활 시 힌트는 v0.9에서 i18n으로 이관된 키를 참조할 것.
- **모드 A 임시표 음 추가 제외** — 무중복 적용 후 A 보통(둘 다)=28로 충분.
- **오답 가중 재출제 비승격** — 이번 불만은 "무의미한 랜덤 중복"이고 오답 재출제는 "의도적 중복"이라는 별개 기능. 백로그 유지(03-plan §9와 정합). 단 본 설계는 확장 여지를 남김: 향후 재출제 문항에 `retried=true`를 달고 dedup에서 제외하면 됨.

**(5) 적용 후 기대 효과**

| 케이스 | 기대 중복 (현행 → 적용 후) | 연속 중복 |
|---|---|---|
| B 쉬움 | 4.50 → **≈0** (양방향 확장으로 풀 14) | 1.29 → **0** |
| A 쉬움 단일 자리표 (풀 9) | 3.77 → **1.01** (비둘기집 최소치) | 1.00 → **0** |
| 풀 ≥ 14 전 케이스 | → **≈0** (확률적 회피, 시뮬레이션 관측치) | → **0** |

| 전역 설정 | 기기/사용자 선호, 드물게 변경 | 언어, 계이름 표기(notation), 사운드, 볼륨 | 설정 페이지 단일 소스 | O |

| 모드 한정 옵션 | 특정 모드에만 의미 | 음자리표(모드 A) | **모드 A 카드 내부** | O |
| 긴급 토글 | 플레이 중 즉시 필요 | 사운드만 | 앱바 🔊 유지 | (전역과 동일 값) |

- **앱바 언어 토글 제거** — 영속화되면 언어는 사실상 1회 설정이라 앱바 상주 근거 소멸. 라벨 모호성 문제도 함께 해소. 게임 앱바 버튼 3→2개로 오터치 감소.
- **음자리표 → 모드 A 카드 내부 이동** — 적용 범위가 배치로 자명해져 각주(`clefAppliesToModeA`) 삭제. 대안(모드 진입 전 옵션 화면)은 매 시작 클릭 +1이라 기각.
- 플레이 옵션이 카드 아래 배치되는 발견성 문제는 **수용** — 영속화로 재방문 시 재설정이 불필요해져 발견성 요구 자체가 하락.

**(2) `src/storage.js` — localStorage 최소 래퍼** (03-plan §9 백로그의 **부분 선행 승격**: 설정만 승격하고 진행 저장 F-19는 계속 백로그. 02-design §10.4는 영속 대상을 4키로 예정했으나 "매 세션 재설정" 불만의 실체가 difficulty/total/clef를 포함하므로 7키로 확장)

```js
// src/storage.js — 설정 영속화 최소 래퍼. 실패 시 조용히 무저장(세션 메모리로 정상 동작).
const KEY = 'sing-settings-v1'; // 스키마 버전은 키에 포함. 구조 변경 시 v2 새 키(마이그레이션 없음)
const PICK = ['lang', 'notation', 'audioEnabled', 'volume', 'difficulty', 'total', 'clef'];
const VALID = {
  lang: (v) => v === 'ko' || v === 'en',
  notation: (v) => ['solfege', 'english', 'both'].includes(v),
  audioEnabled: (v) => typeof v === 'boolean',
  volume: (v) => typeof v === 'number' && v >= 0 && v <= 1,
  difficulty: (v) => v === 'easy' || v === 'normal',
  total: (v) => v === 5 || v === 10,
  clef: (v) => ['treble', 'bass', 'both'].includes(v),
};
export function loadSettings() {
  try {
    const parsed = JSON.parse(localStorage.getItem(KEY) || '{}');
    const out = {};
    for (const k of PICK) if (VALID[k](parsed?.[k])) out[k] = parsed[k]; // 항목 단위 화이트리스트
    return out;
  } catch { return {}; } // Safari 프라이빗 모드 SecurityError / JSON 손상 → 기본값 폴백
}
export function saveSettings(state) {
  try { localStorage.setItem(KEY, JSON.stringify(Object.fromEntries(PICK.map((k) => [k, state[k]])))); }
  catch { /* quota/프라이빗 모드 → 무시 */ }
}
export const PERSIST_KEYS = PICK;
```

- **키 정책(전 릴리스 공통)**: 설정은 `sing-settings-v1`. 학습/진행 데이터(v0.9의 learnSeen, 향후 출제 이력·별 해금)는 **별도 키 `sing-progress-v1`** — 02-design §10.4의 "별도 키" 방침과 정합.
- **로드**: `main.js` 부트스트랩에서 `handle()` 전에 `Object.assign(state, loadSettings())` → `setVolume(state.volume)`·`setMuted(!state.audioEnabled)`로 audio 모듈 동기화. **state = SSOT 확정**(B3 해소).
- **저장**: `setState`에 3줄 훅(`PERSIST_KEYS.some((k) => k in patch)`면 `saveSettings`) — pub/sub 아님, 직접 호출. 볼륨은 setState를 우회하므로(B4) 슬라이더 `onchange`(드래그 종료 1회)에서 `saveSettings` 직접 호출.

**(3) notation 설정 UI** — 설정 페이지, 언어 바로 아래(표기 계열 인접 배치). 기존 `segmented()` 재사용, 라벨 자체가 미리보기: `[도레미] [CDE] [도 · C]`. 게임 중 변경도 안전 — 복귀 재렌더 시 `labelFor()`가 재호출되어 즉시 반영되고, 정답 키는 영어 letter/id 고정이라 채점 무영향.

**(4) 설정 복귀 명시화** — ⚙ 클릭 시 `state.settingsReturn = state.route` 기록, 설정의 뒤로는 `location.hash = state.settingsReturn || '#/menu'`. 직접 진입·새로고침에도 항상 앱 내 복귀(§3.1-5 해소). `settingsReturn`은 비영속. 나중에 추가되는 `#/learn/:id`도 자동 커버.

**(5) B1/B2 수정(S-7, 필수)** — 게임 중 앱바 🔊는 **setState 없이** `setMuted` + 아이콘 직접 갱신(재렌더 회피 — 볼륨 슬라이더의 기존 로컬 갱신 선례와 일관, `game.js` 무수정이라 v0.7의 createQuiz 변경과 완전 독립). B2는 타임아웃 콜백(라우팅·정답음 재생 모두)에서 `state.route`가 여전히 해당 모드일 때만 실행하도록 가드.
> *근본 대안(Codex 비평)*: `submit()`이 즉시 `index++`하지 않고 결과만 반환한 뒤 타임아웃에서 `advance()`로 전진하는 엔진 수정이 향후 어떤 setState에도 안전한 근본 해법이다. v0.8 시점엔 v0.7의 createQuiz 변경이 완료된 뒤라 적용 가능하지만, 변경 표면 최소화 원칙에 따라 기본안은 위 회피 방식으로 하고 엔진 분리는 백로그(§7)에 등재한다.

### 4.4 모드별 학습 콘텐츠

**A. 오선 탐색** (`#/learn/clef-position`)
- 음자리표 segmented [높은음|낮은음] — **로컬 변수로 완전 격리**(전역 `setState({clef})` 금지). *근거: 전역 변경 시 사용자의 '둘 다' 설정을 조용히 덮어쓰고, v0.8의 clef 영속화와 결합하면 기기 영구 파괴로 증폭되는 결합 리스크(통합 검증 major). `state.clef === 'both'`로 진입하면 treble 기본 표시.*
- `createStaffSVG(clef)` + 줄5/칸4/가온다 투명 클릭밴드 → 클릭 시 `drawNote(highlight)` + 음 재생 + 「레 · D5 — 넷째 줄」 라벨(삼중 일치). **재생은 반드시 문자열 피치(`playNote(letter + octave)`) 또는 `midiToFreq(midi)` 경유** — `playNote`에 숫자를 넘기면 Hz로 해석되는 B5 버그와 동일한 오용이 됨.
- 줄/칸 칩 버튼을 동일 핸들러의 **1급 경로**로 제공(키보드·스크린리더 접근성). SVG 클릭밴드는 위치당 최대 ~8px(모바일 ~5px)라 48px 터치 타깃 예외로 명시(칩이 규정 충족 경로, 밴드는 시각 탐색 보조).
- 니모닉 카드(clefs.js 4종) + 가온다 C4 details(+듣기). 클릭 범위는 오선 안 9자리+가온다만(덧줄 확장은 시각 밀도 문제로 제외).

**B. 도레미↔CDE 카드** (`#/learn/note-matching`)
- 공유 오선(treble) + 카드 8개(도~시 + 순환 「(도) C5 ↺」) — 클릭 시 오선 갱신 + `playNote`. 「차례로 듣기」= `playSequence(C4..C5)`(미사용 API 첫 활용).
- 고정도(조가 바뀌어도 도=C) + 옥타브 경계(시 다음은 도, 번호 +1) 설명 카드.
- 재생 버튼 disabled 시간은 고정값이 아니라 **시퀀스 길이로 계산**(`(notes.length − 1) × gapMs + durMs` ≈ 2.75초).

**C. 조표 스테퍼** (`#/learn/key-order`)
- ♯/♭ segmented(로컬) + [−/+] 스테퍼 **0..7** — n=0은 「C장조 — 조표 없음」(기준선. `createKeySigSVG(sign, 0, …)`은 이미 안전). [+] 시 조표 n개 SVG 재생성 + **방금 추가된 조표 음 재생** + 누적 칩(마지막 하이라이트, 남은 순서 흐림) + 장조명(`SHARP_KEYS[n]`/`FLAT_KEYS[n]` + "장조").
- 니모닉 카드 + 「플랫은 샤프의 정확한 역순!」 2줄 비교 카드 + 판별 규칙 details.
- **KEY_RULES 이중화 해소(단일 소스 회복)** — 죽은 데이터를 남긴 채 복제하면 이중화 재생산. 방식은 두 가지: ① i18n 키 3종 이관 + keys.js 삭제(설명 카피=i18n 원칙) ② keys.js에 Ko/En 쌍으로 유지(clefs.js 니모닉 패턴과 통일, Codex 권고). L-6 착수 시 확정 — §8 미결 질문 5.

**D. 화음 쌓기** (`#/learn/chord`)
- 루트(7) × 성질(7) segmented(로컬) → `makeChord` → `createChordSVG` + 「3도씩 쌓기」 해설표(루트/3도/5도/7도 + 반음 수).
- 듣기 3종: 쌓아서(`playSequence`) / 한번에(`playChord`) / **메이저 vs 마이너 비교**(동일 루트 연속 재생).
- Baug(겹임시표)는 학습에선 허용 + 「퀴즈에는 안 나와요」 캡션. **단, Baug 구성음 F𝄪의 피치 문자열 `'F##5'`는 현행 `noteToFreq` 정규식(임시표 1글자만 허용, `audio.js:10`)이 throw하므로 재생 불가** — audio.js 정규식을 `([#b]{0,2})`로 확장(+반음 ±2 가산)하거나 `midiToFreq(tone.midi)` 경유로 재생(외부 크로스 체크에서 발견). SVG 렌더는 ♯♯ 지원되어 문제없음.
