## 1. Executive Summary

악보 입문자가 반복적으로 막히는 3개 지점(음자리표별 위치, 계이름↔영어 이름, 조표 순서)을, **설치·로그인 없이** Vercel에 정적 배포되는 무빌드 웹 게임으로 해소한다. 핵심은 **"짧게·자주·소리와 함께"** 학습 루프이며, 정답 시 Web Audio로 실제 음을 들려주어 시각-청각을 동시에 강화한다.

| 기능(모드) | 학습 목표 | 기간 |
|---|---|---|
| 모드 A — 음자리표 음 위치 | 오선 위치 ↔ 계이름/영어 (높은음/낮은음자리표) | 기간 placeholder |
| 모드 B — 계이름 ↔ 영어 매칭 | 도레미 ↔ CDEFGAB 자동화 (고정도) | 기간 placeholder |
| 모드 C — 조표 순서 | 샤프(파도솔레라미시)·플랫(역순) 순서 | 기간 placeholder |

> 한 문제 = 5초 내 미니 사이클, 한 판 10~15문제(2~3분). 외부 의존성 0개, **Vercel 정적 배포(무빌드)**.

## 2. 배경 · 문제정의 · 목표 / 비목표

### 2.1 문제 정의
- **음자리표별 위치 혼동**: 같은 줄/칸이라도 treble/bass에서 계이름이 달라 헷갈린다.
- **이름 체계의 이중성**: 한국식 계이름(도레미…)과 영어 음이름(C D E…)을 즉시 연결하지 못한다.
- **조표 순서 암기 실패**: #·b이 붙는 고정 순서를 외우지 못한다.
- 기존 학습법은 **즉각 피드백이 없고, 소리와 분리되어 있으며, 지루하다** → 학습 중단.

### 2.2 목표 (Goals)
- 3개 주제를 **짧고 반복 가능한 선택형 퀴즈**로 익히게 한다.
- 정답 시 **실제 음 재생**으로 시각-청각 동시 강화.
- **무설치·무로그인** 즉시 실행 경험(URL 접속만으로 시작).
- **한/영 토글**을 이름 체계 학습 자체에 활용.

### 2.3 비목표 (Non-Goals)
| 제외 항목 | 이유 |
|---|---|
| 박자·리듬(음표 길이) | 3개 주제 범위 밖 |
| 음정·화음·음계 심화 이론 | 입문 범위 초과 (YAGNI) |
| 이동도법(movable-do) | **고정도법**으로 확정 |
| 마이크 연주 채점 | 복잡도 과다, 무빌드 원칙 위배 |
| 계정·서버·전역 랭킹·소셜 | 정적 배포 원칙 상충 |
| 오디오 파일 리소스 | Web Audio 합성으로 대체 |

## 4. 도메인 분석 (핵심 음악 이론)

> 표기 규약: 과학적 음높이 표기법(SPN), 가온다 = **C4**, 계이름은 **고정도**. 게임·오디오의 단일 진실 소스.

### 4.1 주제 1 — 오선 · 음자리표
- 오선 = 5줄 + 4칸, 번호는 **아래→위**. 가온다 **C4**는 두 음자리표가 공유.
- 높은음자리표(G clef): **둘째 줄 = G4** 기준.
- 낮은음자리표(F clef): **넷째 줄 = F3** 기준.

| 위치 | 높은음자리표(아래→위) | 니모닉 | 낮은음자리표(아래→위) | 니모닉 |
|---|---|---|---|---|
| 줄 1→5 | E4 G4 B4 D5 F5 | Every Good Boy Does Fine | G2 B2 D3 F3 A3 | Good Boys Do Fine Always |
| 칸 1→4 | F4 A4 C5 E5 | **FACE** | A2 C3 E3 G3 | All Cows Eat Grass |

- 높은음자리표 아래 첫 덧줄 = 낮은음자리표 위 첫 덧줄 = **C4**.

### 4.2 주제 2 — 계이름 ↔ 영어 매핑 (고정도)
| 도 | 레 | 미 | 파 | 솔 | 라 | 시 | (도) |
|---|---|---|---|---|---|---|---|
| C | D | E | F | G | A | B | C |

- 조와 무관하게 **도 = 언제나 C**. 음이름은 C→B 순환, B4 다음 = C5.

### 4.3 주제 3 — 조표 순서
- **샤프(♯) 순서**: F C G D A E B — 파 도 솔 레 라 미 시
- **플랫(♭) 순서**: B E A D G C F — 시 미 라 레 솔 도 파
- **핵심 관계**: 플랫은 샤프의 **정확한 역순**. 인접 조표는 완전5도 간격.
- 빠른 판별(참고): 마지막 샤프의 반음 위 = 으뜸음 / 끝에서 둘째 플랫 = 으뜸음(단, B♭ 1개는 F장조 예외).

### 4.4 오디오 (12평균율)
- 공식: `f = 440 × 2^((n − 69) / 12)`, A4 = 440 Hz(기준), C4 ≈ 261.63 Hz.
- 계이름→MIDI 매핑 테이블 하나면 충분. 예) C4=60/261.63, G4=67/392.00, C5=72/523.25.

## 7. 게임화 (학습 메커니즘 요약)

### 7.1 공통 코어 루프
```
문제 제시 → 입력 대기 → 판정 → 즉각 피드백(시각+청각) → 점수/콤보 갱신
   → (다음 문제) 반복 → 스테이지 종료 → 결과(별/정확도)
```
상태 뼈대는 3모드 공유, 차이는 `awaitInput`·`judge`의 내용(버튼/오선 클릭/카드/조표 배치)뿐 → 코드 재사용.

### 7.2 점수 · 동기부여
| 요소 | 규칙 | 목적 |
|---|---|---|
| 기본 점수 | 정답 +100 | 성취 |
| 콤보 보너스 | 연속 N개 → +`min(N,5)×20`, 오답 시 리셋 | 몰입 |
| 속도 보너스(선택) | 3초 내 정답 +30 | 숙달 유도 |
| 오답 | **감점 없음**(입문자 좌절 방지), 콤보 리셋 + 정확도 반영 | 심리적 안전 |
| 별 1~3개 | 세션 정확도 ≥70%/≥85%/100% | 성취 시각화 |

> **스테이지 잠금 해제(별 게이팅)는 MVP 제외(백로그)**. 홈은 3개 모드를 잠금 없이 노출.

### 7.3 오답 = 학습 기회 (3단계)
1. **정답 표시**: 정답 초록 하이라이트, 고른 오답 잠깐 붉게.
2. **정답 음 재생**: 맞는 소리를 마지막에 각인. 오답 순간 짧은 버저(`playWrong`) 허용.
3. **니모닉 한 줄**: 예) "높은음자리표 줄은 아래부터 미·솔·시·레·파".
- MVP는 재출제 없이 **결과 화면 오답 리뷰**로 마무리한다(오답 목록은 표시용으로만 기록). 오답 문항의 **스테이지 끝 1회 재출제**는 백로그(§8.2·계획 §9)로 미룬다.

### 7.4 오디오 원칙
- 음원 파일 없음. `OscillatorNode(triangle)` + `GainNode`(attack 5ms / release 150ms).
- "이 위치 = 이 이름 = 이 소리" 삼각 연결. 모드 C는 오디오 비중을 A·B보다 낮게.

### 7.5 모드별 특징 요약
| 모드 | MVP 대표 플레이 | 확장(백로그) |
|---|---|---|
| A | 읽기(4지선다) | 쓰기(오선 클릭, F-04) |
| B | 즉답(4지선다, 방향 랜덤) | 타임어택(타이머), 카드 매칭(메모리) |
| C | 다음 하나 / 순서 퀴즈 | 전체 배치(칩 배열) |

> MVP 난이도 레버(KISS): ①보기 개수(4→7) ②범위 확장 ③자리표/조표 혼합. 새 콘텐츠보다 **기존 요소 조합**으로 난이도 생성. 타이머 ON/OFF·카드 매칭·전체 배치는 §8.2 백로그 참조.

| **오답 1회 재출제** | 결과 화면 오답 리뷰로 충분, 엔진 단순화(§7.3) |
| B 카드 매칭(메모리) / B·C 타이머 변형 / C 전체 배치 | 확장 게임플레이(§7.5), 엔진·상호작용 복잡도 ↑ |
| **별 해금 게이팅** | 진행 저장 의존, 백로그로 이동 |
| 랭킹·계정·대시보드 | 정적/무빌드 원칙 및 YAGNI 위배 |
| 세분화 난이도 | 과설계, 고정 난이도 출시 후 데이터로 판단 |

> 설정(⚙) 화면(한/영·사운드 토글)은 F-16/F-17을 담는 최소 형태로 MVP에 포함(설계 문서 참조).

## 9. 성공 지표 (Success Metrics)

> **MVP는 세션 단위 측정만.** 서버·영속 저장 없이 현재 세션 상태에서 직접 산출. 개인정보·서버 수집 없음.

### 9.1 MVP 지표 (세션 단위)

| 지표 | 정의 | 목표(가설) | 측정 |
|---|---|---|---|
| **정답률** | 정답 수 / 시도 수 (현재 세션) | 세션 반복 시 80% 도달 | 판 종료 시 집계 |
| **세션 내 최고 streak** | 한 세션 내 연속 정답 최대 | 반복 시 상승 추세 | 실시간 카운트 |
| **세션당 문제 수** | 한 세션 풀이 문항 수 | 자투리 학습 성립(≥10) | 세션 카운트 |

> 핵심 선행 지표는 **정답률**과 **세션 내 최고 streak**. 세션 안에서 두 값이 안정적으로 나타나면 학습 루프가 작동한다고 본다.

### 9.2 후순위 지표 (세션 간 영속 저장 필요 — 백로그)

- **주제 숙련 도달**(주제별 최근 정답률 누적), **재도전율**, **최고 기록 추이** 등은 localStorage(F-19) 도입 후 측정. MVP 범위 밖.

> §8.2(저장 MVP 제외)와 정합: 영속 저장이 필요한 지표는 모두 후순위로 분리했다.

### 요약
3개 학습 주제를 각각 **선택형 퀴즈 1종씩**으로 성립시키고, **정답음·즉시 피드백·연속 정답·한영 토글**을 더한 것이 MVP다. 성공 측정은 **세션 단위**(정답률·세션 내 최고 streak·세션 문제 수)로 한정하고, 영속 저장·별 해금·랭킹은 백로그로 미룬다. 배포는 **Vercel 정적(무빌드)**, 도메인 사실(음자리표 배치·계이름 매핑·조표 순서·주파수)은 표준 레퍼런스를 단일 진실 소스로 삼는다.
